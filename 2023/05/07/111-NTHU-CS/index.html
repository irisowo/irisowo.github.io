  1. Which of the following web application attacks are caused by compromising a vulnerable website and uploading malicious code or using malicious links to get a user’s sensitive information?
    A. CSRF attack
    B. XSS attack
    C. Injection attack
    D. Defacing attack

    sol

    B

    CSRF

    CSRF = Cross Site Request Forgery

    • 重點
      • 攻擊者跨 Domain 向網站提出請求, 且網站未經驗證即無條件信任
        帶有此 Cookie 的 request
    • 過程
      1. User 登入 Web A 取得 Cookie ⇒ 存在 User’s browser
      2. User 未登出 Web A 時瀏覽 B 網, Hacker 以 A 網給 User 的 Cookie對 A 網站發送請求, domain 為 B
        e.g., A/pay?to=B means that Web A will do the action “pay” to B domain
    • **CSRF 攻擊三要素
      1. 觸發惡意腳本的動作, e.g., 點到廣告
      2. 只以單一條件驗證網站身份, e.g, Web A’s server 只驗證 cookie 或 token
      3. 沒有驗證(亂數) or 驗證的參數很好猜（固定的 cookie ）
    • 防範
      1. 「SameSite」 要求 Request 來源 == 發布 Cookie 的來源
      2. 增加無法預測的參數例如：CSRF token
      • CSRF token
        Step
        建立 Server 在 User 訪問時生成 Token, e.g., $f$(Username, random, time, 加密) →Token…放在頁面中(注意 Token 不能依然存放在 Cookie 中)
        傳送請求 在添加 Token 的頁面提交請求時，必須帶此 Token ，請求才會被允許 e.g., for GET ⇒ http://url?csrftoken=tokenvalue, for POST ⇒ <input type='hidden' name='csrftoken' value=$tokenvalue/>
        驗證 Server 解密 Token，對比 $f$(Username, random, time, 加密) ，如果加密的資料一致且時間戳記沒有過期，Server 才驗證通過
      1. 增加「驗證碼」機制
    XSS

    XSS = Cross Site Scripting

    • 過程
      1. Hacker 在受害網站注入 XSS 漏洞
      2. 透過社交工程手法傳送惡意 URL 給 User
      3. Required fishing : 當 User 點擊 URL 便會把自己的資料（cookie）藉由受害網站傳回給 Hacker
    • 三大類 XSS
      原理 Required fishing 利用漏洞
      反射型 XSS (Reflected) - 最常見 將惡意 code放進 GET 參數傳遞 http://domain/upload.asp?id=<script>alert(1);</script> (Hacker 通常使用短網址或 HTML Encoder 包裝 URL) O 後端 server
      DOM 型 XSS Hacker 在 URL 輸入 DOM 物件，把物件嵌入網頁程式碼 <img src=#onerror=”alert(123)”> O 前端 js (提取執行皆由 browser 完成)
      儲存型 XSS (Stored) Hacker 將 Javascript 儲存在伺服器的資料庫中 e.g., 留言板注入 js, 下一位User 在網頁載入留言板 js 時受到攻擊, e.g., 留言～<script>alert(1);</script> X 後端 server
    • 防範
      1. 欄位輸入的驗證與檢查, e.g.,URL、檔案上傳、表單欄位、留言
      2. 文法語意檢查：檢查網頁表單輸入欄位的資料類型與內容, e.g., age=^([1-9][0-9]|[1][1-9][1-9])$
      3. 輸入編碼成為純字符串再處理，防止內容被當作程式碼執行
      4. Never add comment, attribute name, label name to input. 因為這些位置都能將字符串作為程式碼運行
      5. 伺服器設定：
        • 對 cookie 增加 HttpOnly 屬性 ⇒ 禁止JavaScript 直接存取 cookie
          (e.g., document.cookie 無法執行取得 cookie)
          確保程式碼沒有存取權設定內容安全策略(CSP)的header
        • 明確定義允許瀏覽器在該頁面上加載的內容來源(e.g., 類型= JavaScriptCSS、HTML框架、字體、圖片)
    Injection attack

    Injection attack

    • 常見的 injection
      • SQL Injection
      • Command injection
        • 例如某網站的查詢欄位可以查詢資料，Hacker 在查詢的內容中打上 hello.com;pwd ，由於Linux可以利用分號來同時提交多個不同的指令, 如果返回結果出現當前目錄 “/var/www“ 則 Hacker 得知 server 為 Linux 系統
    Defacing attack

    Defacing(污損) attack

    • 重點
      • 竄改網頁外觀(通常是政治目的)
    • 案例
      • The Federal Depository Library Program’s website was defaced and showed the following image of President Trump with a vengeful message.

    D ⇒ DoS

    DoS = Denial-of-Service attack

    • 重點
      • Meant to shut down a machine or network, making it inaccessible by flooding and crashing the services.
    • 方法
      1. Buffer overflow attacks
        • ICMP flood
          • review ICMP : a network layer protocol (not transport) to
          • (1) Send a spoofed echo request with spoofed src IP(victim)
            (target = broadcast?)
            (2) The request is transmitted to all of the network hosts on the network ⇒ each host sends an ICMP response to the spoofed source address.
        • SYNK flood
          • sends a request to connect to a server, but never completes the handshakes ⇒ open ports buffer are saturated with requests ⇒ users can’t connect to the server

  14. Which of the following would be needed to block excessive traffic from a particular protocol?
    A. Flood guard
    B. Loop protection
    C. ACL
    D. 802.1X

    sol

    A

    Loop protection

    • 前情提要
      • 網路拓墣中有 cycle, 導致 linking layer 發 broadcast(ARP…) 時出現 broadcast storm
      • 解法是利用 STP(spanning tree protocol)
        • switch 間交換 BPDU 封包
        • 選舉出 root switch, 其餘 switch 各選出一個 root port
        • 每個網段只選一個 designated port
        • block 剩下所有 port
    • STP loop protection
      • If the ports cannot receive BPDUs from the upstream switching device due to link congestion(雍塞) or unidirectional link failures, the switching device will re-select a root port.
        The original root port becomes a designated port, and the original blocked port changes to the Forwarding state, which may cause loops on the network.
        ⇒ If a port enabled, the port will not change to Forwarding state but remain in the Discarding state. In this manner, loops can be prevented.
    802.1x
    • 簡介
      • 802.1X defines a network authentication protocol that opens ports for network access with the help of authentication server.
        (註 : 802.1x 是IEEE制定的一項身分驗證標準，非單一的網路通訊協定)
    • 架構
      • 分三層：申請者(Supplicant), 驗證者(Authenticator), and 驗證伺服器(Authentication Server)，各層之間皆透過 EAP 溝通
        (註 : EAP=Extensible Authentication Protocol=可延伸身份認證協定。依加密方式可細分為EAP-MD5、EAP-TLS、EAP-TTLS、EAP-LEAP，以及安全強化後得EAP-PEAP/PEAP=Protected Extensible Authentication Protocol)

  23. Which of the following describes a network device that intercepts user or host requests and then makes those requests to other hosts or networks on behalf of the user?

    B. Firewall
    C. NIDS
    D. NIPS

    sol

    A
    Both NIDS and NIPS can use signature-based detection

    • NIDS :
      • Monitor traffic that comes in, and looks for suspicious packets based upon the signatures.
      • Detect the threat if a suspicious packet matches up to a signature.
    • NIPS :
      • Monitors and detects just like NIDS
      • Take appropriate follow-up action to mitigate the threat.
    • NIDS/NIPS detection type
      • Signature-based detection
        • monitor traffic and compares it with the preconfigured signatures so as to find a match.
      • Anomaly-based detection
        • Creates a baseline on average network conditions, if any activity outside the baseline parameters is detected, NIPS takes the necessary action.
      • Protocol state analysis detection
        • Identifies deviations of protocol states by comparing observed events with predefined profiles.

  36. Which of the following actions may be considered illegal, depending upon where they take place? (Choose two.)
    A. Wardriving
    B. Warchalking
    C. Jamming attacks
    D. Deauthentication attacks

    sol

    C, D

    • Wi-Fi deauthentication attack
      • A type of DoS targets communication between a user and a Wi-Fi AP.
      • 原理
        • An attacker can send a deauthentication frame(defined in IEEE 802.11 wifi protocol) to a wireless access point, with a spoofed address for the victim ⇒ the victim is deauthenticated.
          The protocol does not require any encryption for this frame, even when the session was established with Wired Equivalent Privacy (WEP) for data privacy, the attacker only needs to know the victim’s MAC address.
      • Usage
        • Evil twin access points
          • The attacker conducts a deauthentication attack to the target client, disconnecting it from its current network, forcing the client to connect to the evil twin access point, which then can be used to capture network packets.
        • Password attack
          • To mount a brute-force or dictionary based WPA password cracking attack on a WiFi user with WPA or WPA2 enabled, a hacker must first sniff the WPA 4-way handshake.
            The user can be elicited to provide this sequence by first forcing them offline with the deauthentication attack.
          • In a similar phishing style attack without password cracking, wifi phisher starts with a deauthentication attack to disconnect the user from their legitimate base station, then mounts a man-in-the-middle attack to collect passwords supplied by an unwitting user.

  39. Which of the following describes an attack in which an attacker captures credentials and transmits them to another host for authentication?
    A. Replay attack

  40. All of the following are valid security issues that allow attacks on WEP, except:
    A. 24-bit initialization vectors
    B. Implementation of RC4
    C. Repeated keys
    D. Use of AES

    sol

  41. Which of the following is an attack vector on networks that use WPA or WPA2?
    A. Use of RC4
    B. Weak passphrases
    C. 24-bit initialization vectors
    D. Use of AES

    sol

    B

    • WEP(Wired Equivalent Privacy 希望跟有線ㄧ樣安全)

      • 加密法 : RC4(Rivest Cipher)

        • $f$(24-bit Initialization Vector, Static 40/104 bit WEP Key)
          ⇒ 64/128 bits RC4加密值
          ⇒ Easy to cracked if we’ve collected enough packets.

      • Integrity Check : CRC-32 checksum (Cyclic Redundancy Check)

        • 線性雜湊演算法
          ⇒ Easy to adjust the checksum so that the resulting message appears valid.
    • WPA
      • 加密法：TKIP
    • WPA2
      • 加密法 : AES

  42. Which of the following attacks enables a malicious person to steal data via Bluetooth devices?

      • Rogue AP
        • A wireless access point that are not part of the network.
        • A rogue access point provides a wireless backdoor channel into the private network for outsiders who can bypass firewalls.
        • 案例

            • May use wifi-deauthentication to force users to connect to the evil AP.
